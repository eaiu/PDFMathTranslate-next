    翻訳サービスドキュメント

    高度な設定 > 翻訳サービスドキュメント (現在のページ)

    コマンドラインで利用可能な翻訳サービスを表示

    コマンドラインでヘルプメッセージを表示することで、利用可能な翻訳サービスとその使い方を確認できます。

    pdf2zh_next -h

    OUTPUT

    ヘルプメッセージの最後で、さまざまな翻訳サービスの詳細情報を確認できます。

    ORIGINAL TEXT

    • Tier 1 (Official Support): - Google Translate: Supports 100+ languages, but the translation quality of some languages is not very good.

    • DeepL Translate: Supports 31 languages, with high translation quality, but the free version has usage frequency limits.

    Tier 2 (Open Source / Self-Hosted)

    • Argos Translate: Supports 100+ languages, but the translation quality is not as good as Google Translate.
    • LibreTranslate: An open-source translation API that supports 100+ languages, but the translation quality is not as good as Google Translate.

    Tier 3 (Deprecated / Not Recommended)

    • Bing Translator: Microsoft's translation service, supports 100+ languages, but the translation quality is not as good as Google Translate.
    • Baidu Translate: Baidu's translation service, supports 200+ languages, but the translation quality is not as good as Google Translate.
    • Youdao Translate: Youdao's translation service, supports 100+ languages, but the translation quality is not as good as Google Translate.

    OUTPUT

    レベル 1（公式サポート）

    • Google Translate：100 以上の言語をサポートしていますが、一部の言語の翻訳品質はあまり良くありません。
    • DeepL Translate：31 言語をサポートしており、翻訳品質は高いですが、無料版には使用頻度制限があります。

    レベル 2（オープンソース / セルフホスト）

    • Argos Translate：100 以上の言語をサポートしていますが、翻訳品質は Google Translate ほど良くありません。
    • LibreTranslate：100 以上の言語をサポートするオープンソースの翻訳 API ですが、翻訳品質は Google Translate ほど良くありません。

    レベル 3（非推奨 / 推奨されません）

    • Bing Translator：Microsoft の翻訳サービスで、100 以上の言語をサポートしていますが、翻訳品質は Google Translate ほど良くありません。
    • Baidu Translate：Baidu の翻訳サービスで、200 以上の言語をサポートしていますが、翻訳品質は Google Translate ほど良くありません。
    • Youdao Translate：Youdao の翻訳サービスで、100 以上の言語をサポートしていますが、翻訳品質は Google Translate ほど良くありません。

    pdf2zh-next supports three types of translation engines: API-based, local models, and browser extensions.

    • API-based: Includes services like DeepSeek, DeepL, Google, etc. These require an API key for use.
    • Local models: Currently supports the Ollama platform, which allows for the deployment of open-source translation models locally.
    • Browser extensions: Currently supports the Immersive Translate browser extension, which enables translation via the extension.

    Note

    If you are using a browser extension, please ensure that the extension is installed and configured correctly.

    After configuring the translation engine, you can select the desired translation engine in the settings.

    Let's begin configuring the translation engine.

    API-based Translation Service Configuration

    DeepSeek

    1. Obtain an API key from DeepSeek Official Website.
    2. Enter the API key in the settings page.

    DeepL

    1. Obtain an API key from DeepL Official Website.
    2. Enter the API key in the settings page.

    Google

    1. Obtain an API key from Google Cloud.
    2. Enter the API key in the settings page.

    Tip

    If you are using Google's free translation API, please note that there may be usage limits.

    Local Model Configuration (Ollama)

    1. Install Ollama: Please follow the Ollama Official Documentation to install Ollama.
    2. Run the model: Use the command ollama run <model_name> to start the model. For example: ollama run translate.
    3. Configure in pdf2zh-next: In the settings page, fill in the Ollama service address (default is http://localhost:11434).

    Note

    Please ensure that the Ollama service is running and accessible.

    Browser Extension Configuration (Immersive Translate)

    1. Install the Immersive Translate browser extension.
    2. Open the extension and configure the translation service.
    3. In pdf2zh-next settings, select the Immersive Translate option.

    Note

    Please ensure that the browser extension is installed and the translation service is configured correctly.

    Testing the Translation Engine

    After configuration, you can use the "Test" button in the settings to verify if the translation engine is working properly.

    FAQ

    Q: Why can't I connect to the Ollama service?
    A: Please check if the Ollama service is running and if the service address is correct.

    Q: Why does the API key not work?
    A: Please check if the API key is correct and has sufficient balance or permissions.

    Q: Why does the translation fail?
    A: Please check the network connection and the status of the translation service.
